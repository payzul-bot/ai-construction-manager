# Режим разработки — Фаза 1: Скелет платформы и визуализация

## Цель текущей фазы

Текущая фаза разработки направлена на построение **полного структурного скелета платформы**
и его **визуализацию во фронтенде** до реализации расчётной логики.

Основная цель фазы:
- увидеть всю структуру платформы целиком;
- проверить иерархии, навигацию и UX;
- убедиться, что архитектура масштабируется;
- зафиксировать структуру до наполнения нормативными данными.

> Примечание по фазам: этот документ описывает **локальный режим разработки** (скелет и визуализация).
> Он **предшествует** Roadmap Phase 1 (MVP Core) и не совпадает с ним по содержанию.

## Что должно быть реализовано в этой фазе

1. Полные иерархические деревья:
   - виды работ (от верхнего уровня до Leaf-уровня);
   - география (страна → регион → город);
   - типы заказчиков (частный / коммерческий / государственный);
   - классы объекта / уровня работ (эконом / комфорт / бизнес / премиум).

2. Фронтенд-обозреватель структуры:
   - навигация по деревьям;
   - возможность проваливаться вглубь и возвращаться назад;
   - breadcrumbs для ориентации в структуре;
   - визуальное различие между узлами и Leaf-элементами.

3. Поведение Leaf-элементов:
   - все Leaf-элементы должны быть видны в интерфейсе;
   - в рамках данной фазы допускается полная реализация **только одного Leaf**;
   - все остальные Leaf должны отображаться со статусом **«В разработке»**.

4. Правила доступа к расчёту:
   - действие «Рассчитать» доступно только если:
     - Leaf реализован;
     - существует соответствующий CalculationProfile и актуальный RulePack;
   - во всех остальных случаях выводится детерминированное сообщение
     «В разработке».

## Что явно запрещено в этой фазе

- Реализация расчётной логики для нескольких Leaf;
- Придумывание или аппроксимация нормативов;
- Хардкод видов работ, регионов или классов во фронтенде;
- Связывание UI-логики с конкретными видами работ;
- Изменение контрактов ядра расчёта.

## Источник истины

Данный документ является явным описанием режима разработки.
При конфликте между реализацией и этим документом
приоритет имеет **этот документ** для Фазы 1.
